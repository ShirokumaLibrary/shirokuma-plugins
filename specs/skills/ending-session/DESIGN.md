# ending-session 設計メモ

スキル管理用メタデータ。実行時には読み込まれない。

## コンセプト

セッション終了時にセッションコンテキストを保存し、Issue ステータスを自動更新する。

### 段階的移行設計

| ステージ | 範囲 | プライマリ記録 | CLI 動作 |
|---------|------|-------------|---------|
| Stage 1（完了: #1181） | スキル・ルール変更のみ | Issue コメント（セッションサマリー） | Handovers Discussion を作成（冗長バックアップ） |
| Stage 2（完了: #1184） | CLI 最適化 | Issue コメント | `--issue-comment` + `--no-handover` で CLI ネイティブ投稿、Handovers 作成はオプション |

### コンテキスト保存先

| セッションモード | プライマリ | セカンダリ |
|----------------|----------|----------|
| Issue バウンド（`#N`） | Issue コメント | Handovers Discussion（Stage 1 のみ） |
| アンバウンド | Handovers Discussion | — |

### サマリー役割分担

`working-on-issue` と `ending-session` の両方が Issue コメントに投稿するが、役割が明確に分かれる:

| スキル | サマリー種別 | 内容 | 投稿条件 |
|--------|-----------|------|---------|
| `working-on-issue` | 作業サマリー | 技術詳細: 変更内容、変更ファイル、PR、技術的判断 | チェーン完了時（自動） |
| `ending-session` | セッションサマリー | メタ情報: 横断的決定、ブロッカー、次のステップ | セッション終了時 |

`ending-session` は作業サマリーの技術詳細を繰り返さない。

### ステータス自動ルーティング

PR の状態から遷移先を自動判定し、ユーザーの推測を不要にする。

| PR 状態 | 遷移先 |
|---------|--------|
| マージ済み | Done |
| オープン | Review |
| なし | 変更なし |

### 信頼性設計

- **プリフライト**: `session preflight` 1 コマンドで全状態（ブランチ、未コミット変更、未プッシュコミット、Issue、PR、バックアップ）を一括取得
- **ローカルフォールバック**: CLI 失敗時に `.claude/sessions/` にローカル保存
- **冪等性**: 他スキルが既にステータス更新済みでもエラーにならない

## トリガーキーワード

SKILL.md の frontmatter `description` フィールドに定義。スキル選択はこのフィールドでマッチングされる。

- "セッション終了", "作業終了", "end session", "引き継ぎ保存"
