# managing-rules 設計メモ

スキル管理用メタデータ。実行時には読み込まれない。

## コンセプト

3 層のルール構造を管理し、プロジェクト固有の規約をパス指定で自動ロードする。

| 層 | ディレクトリ | 管理 |
|----|------------|------|
| プラグインルール | `.claude/rules/shirokuma/` | 自動デプロイ（編集禁止） |
| プロジェクトルール | `.claude/rules/` | ユーザー管理 |
| パス固有ルール | `.claude/rules/` (frontmatter `paths:`) | ユーザー管理 |

### Skills vs Rules の判断基準

- **Skills**: 再利用可能、プラグインで配布、明示的にトリガー
- **Rules**: プロジェクト固有、パスマッチで自動ロード、暗黙的に適用

## トリガーキーワード

SKILL.md の frontmatter `description` フィールドに定義。スキル選択はこのフィールドでマッチングされる。

- "create rule", "add rule", "ルール作成", "ルール追加"
- パス固有の規約設定
