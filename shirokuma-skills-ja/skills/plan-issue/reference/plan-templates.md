# 計画テンプレート

Issue の計画レベルに応じたテンプレート。ステップ 3 で判定したレベルに合うテンプレートを使用する。

## comment-link 構造について

計画の詳細はコメントとして投稿し、Issue 本文にはサマリーリンクを書き込む（comment-link 方式）。

- **コメント（詳細）**: 計画のフル内容（アプローチ・変更ファイル・タスク分解・リスク等）
- **本文（サマリーリンク）**: 計画コメントへのリンク + 1-2行のアプローチ要約

### 本文サマリーリンクのフォーマット

```markdown
## 計画

> 詳細: {コメントURL}

### アプローチ
{1-2行で方針を記載}
```

`{コメントURL}` は `shirokuma-docs items add comment` の返却値 `comment_url` を使用する。

---

## 軽量計画（コメント内容）

```markdown
## 計画

### アプローチ
{1-2行で方針を記載}
```

軽量計画でもコメント投稿は必須。本文にはサマリーリンク（`> 詳細: {url}`）を書き込む。

## 標準計画（コメント内容）

> タスク間に依存関係がある場合、`github-writing-style` ルールの Mermaid ガイドラインに従い図を含める。

```markdown
## 計画

### アプローチ
{選択したアプローチと理由}

### 変更対象ファイル
- `path/to/file.ts` - {変更内容の要約}

### タスク分解
- [ ] タスク 1
- [ ] タスク 2
```

## 詳細計画（コメント内容）

> `github-writing-style` ルールの Mermaid ガイドラインに従い、タスク依存関係・状態遷移・コンポーネント間のやりとりがある場合は図を含める。

```markdown
## 計画

### アプローチ
{複数案の比較と選定理由}

### 変更対象ファイル
- `path/to/file.ts` - {変更内容の要約}

### タスク分解
- [ ] タスク 1
- [ ] タスク 2

### リスク・懸念
- {破壊的変更、パフォーマンス、セキュリティ等}
```

## エピック計画（サブ Issue を持つ Issue の場合）

`subIssuesSummary.total > 0` の Issue では、サブ Issue 構成と integration ブランチを含む拡張テンプレートを使用する。

> `github-writing-style` ルールの Mermaid ガイドラインに従い、サブ Issue 間の依存関係や実行順序を図で表現する。

```markdown
## 計画

### アプローチ
{全体方針}

### Integration ブランチ
`epic/{number}-{slug}`

### サブ Issue 構成

| # | Issue | 内容 | 依存 | サイズ |
|---|-------|------|------|--------|
| 1 | #{sub1} | {概要} | — | S |
| 2 | #{sub2} | {概要} | #{sub1} | M |

### 実行順序
{依存関係に基づく推奨順序}

### タスク分解
- [ ] Integration ブランチ作成
- [ ] #{sub1}: {タスク概要}
- [ ] #{sub2}: {タスク概要}
- [ ] 最終 PR: integration → develop

### リスク・懸念
- {サブ Issue 間の依存リスク}
```

詳細は `epic-workflow` リファレンス参照。
