# ADR テンプレート

## 標準 ADR

Michael Nygard の原典フォーマットに MADR の拡張を加えたもの。

```markdown
**ステータス:** {Proposed | Accepted | Deprecated | Superseded}
**日付:** {YYYY-MM-DD}
[**置換元:** ADR-{NNN}]
[**置換先:** ADR-{NNN}]

## コンテキスト

{この決定・変更の動機となっている問題は何か？
技術的・ビジネス的・社会的要因、プロジェクトの制約を記述する。}

## 決定

{何を提案・実行するのか？
決定を明確かつ簡潔に述べる。}

## 検討した代替案

### {代替案 1}
- **利点:** {メリット}
- **欠点:** {デメリット}

### {代替案 2}
- **利点:** {メリット}
- **欠点:** {デメリット}

[### {代替案 N}
- **利点:** {メリット}
- **欠点:** {デメリット}]

## 結果

### ポジティブ
- {メリット 1}
- {メリット 2}

### ネガティブ
- {トレードオフ 1}
- {トレードオフ 2}

[### リスク
- {リスクと緩和策}]

## 関連する決定
- {ADR-NNN: 関連の概要}
```

## 軽量 ADR

小規模でリスクが低く、容易に可逆な決定向け。

```markdown
**ステータス:** {Proposed | Accepted}
**日付:** {YYYY-MM-DD}

## コンテキスト

{問題・ニーズの簡潔な説明 — 1-3 文。}

## 決定

{決定内容 — 1-2 文。}

## 結果

- {主な結果 1}
- {主な結果 2}
```

## テンプレート選択ガイド

| 基準 | 標準 | 軽量 |
|------|------|------|
| 可逆性 | 元に戻しにくい | 容易に戻せる |
| 影響範囲 | 複数コンポーネント | 単一コンポーネント |
| 代替案 | 2 つ以上検討 | 明白な選択 |
| 関係者 | 複数チーム | 単一チーム・個人 |
| リスクレベル | 中〜高 | 低 |

## 記述のヒント

### コンテキストセクション
- 解決策ではなく問題を記述する
- 関連する制約（タイムライン、予算、チームスキル）を含める
- 関連 Issue や Discussion を参照する

### 決定セクション
- 能動態で記述する: 「X を使用する」（「X が選ばれた」ではない）
- 具体的に: 「PostgreSQL 16」（「リレーショナルデータベース」ではない）
- スコープを含める: 何に適用され、何に適用されないか

### 検討した代替案セクション
- 標準 ADR では最低 2 つの代替案を含める
- 「何もしない」は有効な代替案
- 利点・欠点を公平に記載する — ストローマン的な代替案を避ける

### 結果セクション
- ポジティブとネガティブの結果を分離する
- トレードオフを正直に記載する
- 運用面の影響（デプロイ、監視、保守）を含める
