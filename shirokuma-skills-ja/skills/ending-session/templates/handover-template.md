# 引き継ぎテンプレート

引き継ぎ Discussion 投稿を作成する際のフォーマット。

## テンプレート

```markdown
## サマリー
{このセッションで達成した内容を1-2文で}

## 関連 Issue
- #{issue_number} - {状態: started/in-progress/completed/blocked}

## 主要な決定事項
- {決定1: 根拠を添えて}
- {決定2: 根拠を添えて}

## ブロッカー
- {ブロッカーの説明と解消に必要なこと}
- または: なし

## 次のステップ
- [ ] {具体的なアクション1}
- [ ] {具体的なアクション2}
- [ ] {具体的なアクション3}

## 変更ファイル
- `path/to/file1.ts` - {変更内容}
- `path/to/file2.tsx` - {変更内容}

## メモ
{次のセッションに必要な補足情報やリンク}
```

## 例

```markdown
## サマリー
ユーザー認証フォームをバリデーションとエラーハンドリング付きで実装した。

## 関連 Issue
- #123 - in-progress（フォーム完了、API 連携が必要）
- #456 - started（要件を確認）

## 主要な決定事項
- React Hook Form でフォーム状態管理（TypeScript サポートが優れている）
- JWT トークンを httpOnly Cookie に保存（セキュリティ）

## ブロッカー
- バックエンドチームからの API 仕様待ち（@backend-team）

## 次のステップ
- [ ] API 仕様が確定次第、認証 API と連携
- [ ] フォームバリデーションのユニットテスト追加
- [ ] 「パスワードを忘れた」フロー実装

## 変更ファイル
- `components/auth/login-form.tsx` - ログインフォームコンポーネント新規作成
- `lib/validations/auth.ts` - 認証用 Zod スキーマ
- `messages/ja/auth.json` - 日本語翻訳

## メモ
バックエンドチームによると、API は明日の EOD までに準備予定。
参考デザイン: https://example.com/design
```

## ガイドライン

1. **サマリー**: 活動ではなく成果に焦点を当て、簡潔に
2. **関連 Issue**: 該当する場合は必ず GitHub Issue にリンク
3. **主要な決定事項**: 「何を」ではなく「なぜ」を記録
4. **ブロッカー**: 解消に必要なことを具体的に
5. **次のステップ**: 具体的かつ実行可能に
6. **変更ファイル**: 次のセッションがスコープを把握できるように
7. **メモ**: 他のセクションに収まらない情報を記載
