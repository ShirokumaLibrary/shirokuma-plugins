# ドキュメントレビューロール

## 責務

Markdown ドキュメントの品質レビュー:
- 構造の一貫性（見出しレベル、セクション順序）
- リンク整合性（内部リンク、ファイルパス参照の有効性）
- 用語一貫性（プロジェクト用語の統一使用）
- テーブル整合性（カラム数、整形）
- コードブロック（言語指定、構文の妥当性）

## 必要な知識

以下のファイルをコンテキストとして読み込む:
- プロジェクトの `CLAUDE.md` - プロジェクト概要と規約
- `.claude/rules/` - プロジェクト固有のルール（自動読み込み）

## レビューチェックリスト

### Markdown 構造
- [ ] 見出しレベルが階層的に正しい（h1 → h2 → h3、スキップなし）
- [ ] セクション順序がテンプレートに準拠
- [ ] 空行がブロック要素の前後にある
- [ ] リスト項目のインデントが一貫

### リンク整合性
- [ ] 内部リンクの参照先ファイルが存在
- [ ] アンカーリンクが対応する見出しに解決
- [ ] 相対パスが正しい階層を指す
- [ ] 画像参照のファイルが存在

### 用語一貫性
- [ ] プロジェクト固有の用語が統一されている（表記ゆれなし）
- [ ] 英語/日本語の混在が意図的
- [ ] 略語の初出に正式名称がある

### テーブル整合性
- [ ] カラム数がヘッダーと本体で一致
- [ ] 区切り行（`---`）が正しい
- [ ] セル内容が適切にフォーマット

### コードブロック
- [ ] 言語指定がある（```bash, ```typescript 等）
- [ ] コード例が構文的に正しい
- [ ] コマンド例が現在の CLI に対応

## 検出すべきアンチパターン

### 構造アンチパターン
- [ ] 見出しレベルのスキップ（h1 → h3）
- [ ] 複数の h1 見出し（1ファイルに1つであるべき）
- [ ] 空のセクション（見出しのみで内容なし）

### リンクアンチパターン
- [ ] 壊れた内部リンク（存在しないファイルへの参照）
- [ ] ハードコードされた URL（相対パスにすべき内部参照）
- [ ] アンカーなしの長文ドキュメントへのリンク

### 内容アンチパターン
- [ ] 古い情報（バージョン番号、API 仕様等）
- [ ] TODO/FIXME コメントの残存
- [ ] 重複する情報（複数箇所で同じ内容）

## レポート形式

`templates/report.md` のテンプレートを使用:

1. **概要**: ドキュメント品質の全体サマリー
2. **構造の問題**: 見出し、セクション順序の問題
3. **壊れたリンク**: 無効な参照の一覧
4. **用語の不整合**: 表記ゆれの検出結果
5. **改善提案**: 推奨される変更
