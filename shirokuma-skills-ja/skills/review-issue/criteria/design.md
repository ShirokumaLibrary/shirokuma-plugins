## UI 設計成果物

### Design Brief

| 基準 | 評価観点 |
|------|---------|
| Purpose | 解決する問題が具体的に記述されているか。曖昧な「改善する」ではなく、測定可能な目標があるか |
| Context | 技術スタック制約、既存デザインシステム、ブラウザ要件が網羅されているか |
| Differentiation | 競合・既存UIとの差別化が具体的か。「モダンにする」等の抽象表現ではないか |
| Target Users | ユーザーペルソナまたはロールが明確に定義されているか |

### Aesthetic Direction

| 基準 | 評価観点 |
|------|---------|
| Tone 定義 | 明確なトーン（professional, playful, minimal 等）が選択され、根拠があるか |
| Colors | プライマリ、セカンダリ、アクセント、背景、テキスト色が定義されているか |
| Spacing | 間隔の体系（4px/8px グリッド等）が定義されているか |
| Typography | フォントファミリー、サイズスケール、ウェイトが定義されているか |
| Borders & Shadows | ボーダー半径、シャドウレベルが定義されているか |
| shadcn/ui テーマ整合 | CSS 変数が shadcn/ui のテーマ構造に従っているか |

### UI 実装

| 基準 | 評価観点 |
|------|---------|
| コンポーネント構成 | 単一責任、適切な粒度で分割されているか |
| Tailwind CSS v4 | ユーティリティクラスが適切に使用されているか。`@theme inline` 等の v4 固有パターンに準拠しているか |
| レスポンシブ | ブレークポイント（sm, md, lg, xl）が適切に設定されているか |
| アニメーション | トランジションが 200-300ms の範囲で適切か。過度なアニメーションがないか |
| アクセシビリティ | WCAG 2.1 AA 準拠（コントラスト比 4.5:1 以上、キーボード操作対応、aria 属性） |
| Radix UI ハイドレーション | `mounted` ステートパターンが必要な箇所で使用されているか |
