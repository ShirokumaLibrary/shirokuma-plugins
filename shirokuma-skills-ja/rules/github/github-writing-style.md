# GitHub ライティングスタイル

GitHub 書き込み（Issues, PRs, Discussions の本文・コメント）における記述スタイル。

## 箇条書き vs 散文

| 条件 | 形式 |
|------|------|
| 3+ 項目の並列情報 | 箇条書き |
| 手順・順序あり | 番号付きリスト |
| 因果関係・論理展開 | 散文 |
| 単一の説明文 | 散文 |

## フォーマットルール

- **並列構造の統一**: 各項目を同じ品詞・構造で開始する
- **ネスト上限**: 2段まで（3段以上は分割またはテーブルに変換）
- **リスト長**: 1リストあたり 2-7 項目（8+ はグループ分割を検討）
- **導入文**: リスト前に1文のコンテキストを追加する

## アンチパターン

| パターン | 問題 | 修正 |
|---------|------|------|
| Wall of Bullets | 全文を箇条書き化、要点が埋没 | 散文と箇条書きを適切に使い分ける |
| 並列構造の崩れ | 項目ごとに異なる構文 | 統一した文法構造で書き直す |
| 導入文なしリスト | コンテキスト欠如で意味不明 | リスト前に概要文を1文追加 |
| 単一項目リスト | 箇条書きの意味がない | 散文に変換 |

## スコープ

GitHub Issues / PRs / Discussions の本文・コメントに適用。スキル完了レポートのスタイルは `completion-report-style` ルール、Claude 設定ファイルの構造は `managing-agents/documentation-structure.md` を参照。
