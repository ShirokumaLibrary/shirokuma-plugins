# バッチワークフロー

共通のエリアやテーマを持つ複数の小さい Issue を、1 ブランチ・1 PR で処理する。

## 適格性基準

| 基準 | 要件 |
|------|------|
| サイズ | XS または S のみ（M 以上は個別処理） |
| 関連性 | 同一の `area:*` ラベル、または同一ファイル群に影響 |
| 独立性 | Issue 間にブロッキング依存がない |
| 上限 | 1 バッチ 5 Issue 以下（推奨） |

**バッチ不可:** TDD が必要な Issue（各 Issue に独自のテストサイクルが必要）、異なる `area:*` ラベルかつファイル重複なし。

## ブランチ命名

```
{type}/{issue-numbers}-batch-{slug}
```

type 混在時は `chore` をデフォルト。Issue 番号はハイフン区切り昇順。

## ステータス管理

バッチ開始時に全 Issue を一括で In Progress に移行する（`project-items` ルールの例外）。

## 並列バッチモード（実験的）

> `isolation: worktree` を使用した並列処理。`--parallel` フラグで起動。

完全に独立したファイルセットを操作する Issue 群を、worktree 分離で並列処理する。

### 逐次バッチとの使い分け

| 条件 | モード |
|------|--------|
| 共通ファイルを操作する Issue 群 | 逐次バッチ（1ブランチ・1PR） |
| 完全に独立したファイルセットの Issue 群 | 並列バッチ（各 Issue が独立 PR） |

### 並列バッチ適格性基準

逐次バッチの基準に加え、以下を満たすこと:

| 基準 | 要件 |
|------|------|
| ファイル独立性 | Issue 間で変更ファイルが重複しないこと |
| 同時起動数 | デフォルト 3、最大 5（`--parallel=N` で指定） |

### 集約パターン

各 Issue が独立した PR を作成する（PR ベース集約）。バッチブランチは作成しない。

詳細（ワークフロー・エラーハンドリング・動作確認手順）は `working-on-issue/reference/batch-workflow.md` を参照。
